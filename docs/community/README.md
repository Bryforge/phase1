## Community support goal

Bryforge support should be useful, searchable, safe, and welcoming.

The future forum should help users find answers quickly while keeping maintainers organized and preserving clear boundaries between support, bug reports, feature requests, security reports, roadmap discussion, hardware/recovery support, and language/toolchain discussion.

## Security and privacy rule

Community support must not ask users to post secrets, credentials, private keys, recovery codes, tokens, email passwords, Apple ID credentials, private logs, unrevised screenshots, or sensitive local files.

Security-sensitive issues should route to `SECURITY.md` instead of normal public forum threads.

## Relationship to GitHub

The future forum should complement GitHub:

- forum discussion for questions and triage;
- GitHub issues for confirmed bugs and actionable feature requests;
- GitHub pull requests for implementation;
- docs updates when questions repeat;
- roadmap updates when support themes become planned workstreams.

## Non-claims

This community index does not launch a forum, create guaranteed support, or replace official repository docs, issues, release notes, validation reports, or security reporting paths.
