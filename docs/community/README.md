## Community support goal

Bryforge support should be useful, searchable, safe, and welcoming.

The future forum and support AI should help users find answers quickly while keeping maintainers organized and preserving clear boundaries between support, bug reports, feature requests, security reports, roadmap discussion, hardware/recovery support, and language/toolchain discussion.

## Automated support AI goal

The planned support AI should help users:

- find the right docs quickly;
- understand Phase1, Base1, and Fyr status boundaries;
- run safe read-only diagnostics;
- prepare useful bug reports and support requests;
- avoid posting secrets or private information;
- route security-sensitive reports to the correct private path;
- turn repeated support questions into documentation improvements.

The support AI should not replace maintainers, GitHub issues, `SECURITY.md`, release notes, validation reports, or official repository documentation.

## Security and privacy rule

Community support and automated support must not ask users to post secrets, credentials, private keys, recovery codes, tokens, email passwords, Apple ID credentials, private logs, unrevised screenshots, sensitive local files, or production data.

Security-sensitive issues should route to `SECURITY.md` instead of normal public forum threads or AI-assisted public support flows.

## Relationship to GitHub

The future forum and support AI should complement GitHub:

- forum discussion for questions and triage;
- AI-assisted drafting for support, bug, feature, documentation, and crypto-policy issues;
- GitHub issues for confirmed bugs and actionable feature requests;
- GitHub pull requests for implementation;
- docs updates when questions repeat;
- roadmap updates when support themes become planned workstreams.

## Non-claims

This community index does not launch a forum, deploy automated support AI, create guaranteed support, or replace official repository docs, issues, release notes, validation reports, or security reporting paths.
