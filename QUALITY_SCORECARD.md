# Phase1 Quality Scorecard

The Phase1 quality scorecard is a lightweight repository health check.

It is generated by:

```bash
sh scripts/quality-score.sh
```

## Current scoring areas

| Area | Points | What it checks |
| --- | ---: | --- |
| Required docs | 20 | Project docs and quality docs exist |
| Required scripts | 20 | Quality and project validation scripts exist |
| Rust source | 10 | `src/main.rs` and `Cargo.toml` exist |
| Tests | 15 | Rust tests or test scripts exist |
| CI workflows | 15 | GitHub Actions workflows exist |
| Safety docs | 10 | Security docs exist |
| Release docs | 10 | Update/release docs exist |

## Score interpretation

| Score | Meaning |
| ---: | --- |
| 90-100 | Excellent project hygiene |
| 75-89 | Good, but some infrastructure can improve |
| 50-74 | Risky; missing important quality assets |
| 0-49 | Not release-ready |

## Release target

Phase1 should target **90+** before release.

## Notes

The scorecard intentionally does not replace Rust tests, security review, manual UX review, or release validation. It is a fast signal that the quality system is present and basic repository structure is healthy.

Future scorecard improvements should include:

- test count tracking
- documentation link checking
- release version consistency
- command registry coverage
- security-policy coverage
- terminal/Gina/Base1 specific quality modules
